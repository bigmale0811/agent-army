## 需求描述

使用者有一個虛擬人物角色圖片和一首歌曲（MP3）。希望系統能自動化產出一支完整的 MV 影片，包含：AI 生成的背景、角色合成、對嘴動畫、最終影片輸出，以及 YouTube 上傳用的標題/描述/標籤。

整個流程應該是「一鍵觸發」— 使用者只需提供歌曲資訊和 MP3，系統自動完成所有步驟。

使用者透過 Telegram Bot 操作，傳送 MP3 檔案和歌曲資訊即可觸發管線。

## 功能項目

| # | 功能 | 說明 | 必要/選配 |
|---|------|------|----------|
| F-1 | 歌曲研究 | 用 LLM 分析歌曲風格、情緒、視覺風格，產出背景描述 prompt | 必要 |
| F-2 | 文案產出 | 用 LLM 產出 YouTube 標題、描述、標籤 | 必要 |
| F-3 | AI 背景生成 | 用 ComfyUI（SDXL）根據歌曲風格生成 1920x1080 背景圖 | 必要 |
| F-4 | 角色合成 | 去背（rembg）+ 角色貼合到背景上 | 必要 |
| F-5 | 品質預檢 | 檢查圖片規格、人臉偵測、音訊格式、工具可用性 | 必要 |
| F-6 | MV 影片合成 | 用 SadTalker 產生對嘴動畫，或 FFmpeg 靜態模式作為備援 | 必要 |
| F-7 | Telegram Bot 介面 | 透過 Telegram 傳 MP3 + 歌曲資訊觸發管線，回傳進度和結果 | 必要 |
| F-8 | CLI 介面 | 命令列執行管線，支援所有參數 | 必要 |
| F-9 | 專案管理 | 儲存/讀取/列出所有 MV 專案的狀態和產出物 | 必要 |
| F-10 | Gemini 多模態審查 | 用 Gemini 視覺模型評估角色圖 + 背景的搭配度 | 選配 |
| F-11 | 服裝修改 | 根據歌曲風格修改角色服裝（未來 v2 功能） | 選配/未來 |

## 驗收標準

> 每一條驗收標準都必須可測試。QA Agent 會根據這些標準獨立撰寫測試。

| # | 驗收標準 | 測試方式 |
|---|---------|---------|
| AC-1 | CLI 執行 `python -m src.singer_agent --dry-run --title "測試" --artist "歌手" --audio test.mp3` 可在 10 秒內完成，exit code = 0 | E2E |
| AC-2 | 歌曲研究結果包含 genre、mood、visual_style、color_palette、background_prompt 欄位，均為非空字串 | 單元 |
| AC-3 | 文案產出結果包含 title、description、tags 欄位，tags 為非空 list | 單元 |
| AC-4 | ComfyUI 背景生成成功時回傳 PNG 檔案路徑，該檔案為有效圖片（寬 ≥ 1920、高 ≥ 1080） | 整合 |
| AC-5 | ComfyUI 不可用時，自動降級為純色背景（PIL），仍產出有效圖片 | 單元 |
| AC-6 | 角色去背後為 RGBA 透明背景，合成圖中角色位於底部置中 | 單元 |
| AC-7 | 品質預檢回傳結構化結果，包含 passed/failed 判定和各項目分數 | 單元 |
| AC-8 | MV 合成成功時產出 .mp4 檔案，時長與原 MP3 誤差 ≤ 2 秒 | 整合 |
| AC-9 | SadTalker 不可用時，自動降級為 FFmpeg 靜態模式，仍產出 .mp4 | 單元 |
| AC-10 | Telegram Bot 收到 MP3 + caption 後觸發管線，每步傳送進度訊息 | E2E |
| AC-11 | Telegram Bot 管線完成後傳送影片檔案給使用者 | E2E |
| AC-12 | 所有 8 步管線執行過程中，任何步驟失敗不會閃退，提供清楚錯誤訊息 | E2E |
| AC-13 | 專案 JSON 儲存包含完整狀態，可用 `--list` 列出所有專案 | 單元 |
| AC-14 | 含非 ASCII 路徑的檔案（中文歌名）可正常處理，不報 encoding 錯誤 | 整合 |

## 邊界條件與錯誤處理

| 情境 | 預期行為 |
|------|---------|
| Ollama 離線 | 歌曲研究步驟失敗，提示使用者啟動 Ollama，管線中止但不閃退 |
| ComfyUI 離線 | 自動降級為純色背景，管線繼續 |
| SadTalker 不可用 | 自動降級為 FFmpeg 靜態模式，管線繼續 |
| MP3 檔案損壞 | 品質預檢步驟偵測到，提供錯誤訊息，管線中止 |
| 角色圖片無人臉 | 品質預檢警告，但管線繼續（降級模式） |
| 磁碟空間不足 | 影片合成步驟失敗，提供空間不足提示 |
| 非 ASCII 檔案路徑 | SadTalker 呼叫時複製到暫存 ASCII 路徑，完成後清理 |
| Telegram 未授權使用者 | 拒絕請求，不執行管線 |
| 同時觸發多個管線 | 依序執行（佇列），不並行（避免 GPU 衝突） |
| Gemini API 無 Key | 跳過 Gemini 審查步驟，管線繼續 |

## 管線流程（8 步）

```
Step 1: 歌曲研究（Ollama/LLM）
  ↓
Step 2: 建立歌曲規格（SongSpec）
  ↓
Step 3: 文案產出（Ollama/LLM）
  ↓
Step 4: AI 背景生成（ComfyUI SDXL）
  ↓
Step 5: 角色合成（rembg 去背 + PIL 合成）
  ↓
Step 6: 品質預檢（圖片/音訊/工具檢查）
  ↓
Step 7: MV 影片合成（SadTalker / FFmpeg 備援）
  ↓
Step 8: 完成（儲存專案、通知使用者）
```

## 不做的事（Out of Scope）

- ❌ 不做影片上傳到 YouTube（只產出檔案和文案）
- ❌ 不做即時串流/直播
- ❌ 不做多角色同畫面（一次只有一個角色）
- ❌ 不做 MV 剪輯/轉場（單一連續鏡頭）
- ❌ 不做歌詞字幕疊加（未來版本考慮）
- ❌ 不做服裝修改（F-11 列為未來功能）
