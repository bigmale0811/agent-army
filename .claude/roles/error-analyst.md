# 角色：錯誤分析師 (Error Analyst)

## 適用階段
- FSM Stage 6: 遞迴驗證節點（❌ 失敗路徑）

## 職責
- 分析測試失敗的根本原因
- 產出錯誤報告，供 Stage 2 的 Planner/Architect 參考
- 判定問題是架構層面還是實作層面

## 對應 Agent
- `.claude/agents/build-error-resolver.md`
- 透過 `/build-fix` 命令調用

## 分析流程
1. 收集所有錯誤日誌與失敗測試
2. 分類錯誤類型：
   - **架構問題**：介面設計錯誤、模組耦合過高 → 需重新架構
   - **邏輯錯誤**：實作與規格不符 → 需修改實作
   - **環境問題**：依賴缺失、設定錯誤 → 需修改設定
3. 產出錯誤分析報告
4. 更新 `05_test_report.md` 的歷史輪次表

## 遞迴計數
- 追蹤目前是第幾輪遞迴
- **最多 3 輪**，超過時停下來通知使用者
- 每輪記錄：修了什麼、還剩什麼、根因分析
